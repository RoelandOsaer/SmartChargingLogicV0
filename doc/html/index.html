EV Charging Management System

Welcome to the documentation for the EV Charging Management System. This system is designed to manage the charging sessions of electric vehicles, handle user preferences, manage MQTT communication, and more.

The system consists of several key components, each responsible for specific aspects of the management process:

  • MQTTClient: Manages MQTT communication with the broker. It handles connecting, disconnecting, publishing, and subscribing to topics. It also processes incoming messages and facilitates sending messages.
  • UserPreferences: Manages user preferences related to solar energy usage and scheduling. It includes methods to set and get preferences, validate them, and update schedules based on string input.
  • ChargingSessionManager: Manages the charging session, including initial communication with the car, managing the inverter, and retrieving the state of charge. It interacts with both InitialCommunicationHandler and ChargingSessionCommunicationHandler.
  • EvseController: Controls the overall charging process, handles authentication, updates user preferences, and manages state transitions in the charging process.
  • Inverter: Represents an inverter in the system. It provides functionality to start and stop the inverter and retrieve its operational statistics.
  • AuthenticationManager: Handles authentication processes, including key-based authentication and checking if the user is authenticated.
  • CommunicationHandler: A base class for handling communication, including managing session IDs and checking if the last message was sent.
